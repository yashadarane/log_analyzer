# Log Analyzer Production-Grade Log Processing & Anomaly Detection

A memory-efficient Python CLI tool that processes large log files, computes
structured summary metrics, detects error-rate anomalies, and reports the top
recurring error messages.

## Quick Start

### 1 Generate sample logs (only needed once)
```bash
python generate_sample_logs.py
```

### 2 Run the analyzer
```bash
# Print JSON to terminal
python main.py --logs sample_logs/

# Save JSON to a file
python main.py --logs sample_logs/ --output result.json

# Custom settings
python main.py --logs sample_logs/ --threshold 5 --top-k 10 --window 600

# Verbose mode (shows per-file progress on stderr)
python main.py --logs sample_logs/ --verbose
```

### 3 Run tests
```bash
python -m unittest discover tests/ -v
```

## CLI Options

| Flag            | Default | Description                                       |
|-----------------|---------|---------------------------------------------------|
| `--logs / -l`   | —       | **Required.** Directory containing `*.log` files  |
| `--output / -o` | stdout  | Write JSON to this file instead of printing       |
| `--top-k`       | 5       | Number of top error messages to report            |
| `--window`      | 300     | Sliding window size in seconds (default = 5 min)  |
| `--threshold`   | 10      | Error count per window that triggers an anomaly   |
| `--verbose / -v`| off     | Enable DEBUG-level logging to stderr              |

## Design Approach

### Memory Efficiency — Generator Pattern
The biggest challenge with large log files is memory. Loading a 10 GB file
into a list would crash the program. Instead, every file is read **one line
at a time** using Python generators (`yield`). Memory usage stays constant
(O(1)) regardless of file size.

```
stream_log_directory()
    └── stream_log_file()         ← one line at a time
            └── parse_line()      ← converts text to LogEntry
```

### Single-Pass Analysis
The generator is consumed **exactly once**. While streaming through all log
lines, we simultaneously:
- Count totals (summary metrics)
- Collect ERROR messages into a plain list for top-K analysis
- Collect ERROR timestamps into a list for anomaly detection

Only ERROR entries are kept in memory — not all log entries.

### Sliding Window Anomaly Detection
Time is divided into fixed 5-minute "buckets" (e.g. 10:00–10:05, 10:05–10:10).
Each ERROR timestamp is mapped to its bucket. If any bucket's count exceeds
the threshold, it is flagged as an anomaly.

### Separation of Concerns — Module Responsibilities
| Module        | Does exactly one thing                             |
|---------------|----------------------------------------------------|
| `parser.py`   | Turns raw text lines into `LogEntry` objects       |
| `metrics.py`  | Counts logs, computes rates, finds top-K errors    |
| `anomaly.py`  | Detects anomalous time windows                     |
| `formatter.py`| Converts results to the required JSON structure    |
| `analyzer.py` | Orchestrates the pipeline (imports all the above)  |
| `main.py`     | CLI only — parses args, calls analyzer, prints     |

## Edge Cases Handled

| Situation                         | Behaviour                              |
|-----------------------------------|----------------------------------------|
| Malformed log line                | Skipped with a WARNING log message     |
| Wrong number of `\|` separators   | Skipped with a WARNING log message     |
| Invalid timestamp format          | Skipped with a WARNING log message     |
| Blank / whitespace-only lines     | Silently skipped                       |
| File encoding errors              | Characters replaced, file continues    |
| Missing / unreadable file         | Logged as ERROR, other files continue  |
| Non-existent directory            | Raises `FileNotFoundError` immediately |
| Directory with no `*.log` files   | Returns empty result with a WARNING    |
| Empty log files                   | Processed safely, contribute 0 entries |
| Logs from multiple files out-of-order | Timestamps sorted before anomaly detection |
